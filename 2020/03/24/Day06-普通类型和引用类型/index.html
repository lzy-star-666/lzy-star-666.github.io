第三单元 part2 组合类型

类型传递例子

public class Worker {
    int grade=4;
    String name ="zhangsan";

    public static void main(String[] args) {
        // w1 是一个变量，类型是worker，是符合类型，Worker是它的值；
        Worker w1=new Worker();
        // 类名的首字母大写，变量名的首字母小写（骆驼命名）
        // String，这也是一个引用类型
        int age =21;
    }
}

引用类型会分别在栈和堆中开辟两个内存，当有等号的时候，栈的地址会指向堆的地址。

口述1： 怎样理解组合类型？

口述2：内存的逻辑划分是什么？各部分的作用是什么？

！1.栈：放的是局部变量，用于频繁的数据交换，特点是先进后出。由OS（操作系统）来管理，程序员无法掌控。
！2.堆： 放的对象,new出来的东西放在堆区。
3.全局数据区：全局变量，static变量。
4.代码段：放代码的地方 。
5.内存单元都有唯一编号

口述3：基本类型传递和引用类型值传递的区别？

public class ValueTransfer {
    /**
     * 
     * @param age年龄
     * @return 折半的值
     */
    public int harf(int age) {
        System.out.println(age);//20
        int result = age / 2;
        System.out.println(age);//10
        return result;

    }

    public void change(Worker wx) {
        wx.grade = 100;
        wx.name = "李宁";
    }

    public static void main(String[] args) {
        ValueTransfer valueTransfer = new ValueTransfer();
        int x = 20;
        int result = valueTransfer.harf(20);
        System.out.println(x); // 20
        System.out.println(result);// 10

        Worker xa = new Worker();
        valueTransfer.change(xa);
        System.out.println(xa.grade);
        System.out.println(xa.name);

    }
}

简单类型传递就是值得复制，不会对原值产生影响；
引用类型的值实际上是内存地址，传递的过程就是地址的指向。

口述4：java的垃圾回收机制（GC机制）？

gc：Garbage Collector
垃圾：堆区中不用的对象；
内存泄漏：无用对象还在内存中待着；
自动垃圾回收机制，
面试题：Java有没有可能内存泄漏？
答：有可能的。
*面试题：System.gc()方法能否强迫JVM进行垃圾回收 *
不可以的，gc方法只是建议JVM进行垃圾回收；
gc方法一调用，JVM就立刻工作回收垃圾（×）
Object:finalize()：在正式回收垃圾前它可被调用，做一些收尾的工作；
程序员能否深度干预垃圾回收？ 不可以

独立任务：不用两个中间变量进行数值交换

1、加减法：该方法可以交换整型和浮点型数值的变量，但在处理浮点型的时候有可能出现精度的损失

a = a + b;

b = a - b;

a = a - b;

2、异或法：可以完成对整型变量的交换，对于浮点型变量它无法完成交换。

a = a^b;

b = a^b;

a = a^b;
3、乘除法：可以处理整型和浮点型变量，但在处理浮点型变量时也存在精度损失问题。而且乘除法比加减法要多一条约束：b必不为0。

a = a * b

b = a / b

a = a / b
